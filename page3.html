1.5: Describing copyright and other constraints affecting websites
Copyright in regards to websites is created at the time of development, effectively meaning that the material displayed on the website is not to be used without the permission of the owner. As of 1989, a public display of a copyright is not required to have a copyright take effect. In the UK, it costs £42.50 for 5 years or £72.50 for 10 years of copyright protection. Alongside this fee, you must send in a non-returnable deposit. This deposit contains the material you wish to copyright. This means that any updates to the website after the copyright is secured will require an updated claim with a new fee.[15][16] The GDPR affects the handling of user data, and any goods sold on websites must obey laws regarding general commerce.

1.6: Describing access issues that need taking into account - describe issues that people with disabilities may face when using webpages and how we can overcome them
The issue that requires the least technical knowledge to resolve is to improve accessibility for those with partial or full colour blindness, as all it requires is to avoid putting colours such as blue, green and yellow near each other. For more general colour accessibility, I will use white text on a black background due to personal preference, though the ability to include a light mode with the colour palette swapped will be taken into consideration.
Text to speech can be used to aid the visually impaired, with alt tags on all images so the text-to-speech can describe the image. Subtitles should be available for any videos and transcripts for sound files to prevent the hearing impaired from missing out on any features of my website. For those with motor control issues, the clickable range of links can be increased to make it less difficult to click with a mouse.[17]


1.7: Describe which are the appropriate filetypes for websites
HTML is described as the most basic building block of the Web. It defines the meaning and structure of Web content by using to change the format of the page and create hyperlinks to other pages.[18] CSS is a language used to control the style and feel of the content written in HTML. It does this by creating rules and applying them to various elements in the website. It works by attaching .css commands to HTML tags to change their visual properties.[19] JS is a language that-when used in the context of Web design-allows us to make changes to how certain features on a website function, enabling more advanced prompts such as checkboxes and calls-to-action with .js commands.[20]
